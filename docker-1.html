Instalación de Docker
Ubuntu / Debian
Actualizar repositorios e instalar Docker
# Actualizar lista de paquetes sudo apt update # Instalar Docker sudo apt install docker.io -y
Iniciar el servicio y habilitarlo para que arranque automáticamente con el sistema
sudo systemctl start docker sudo systemctl enable docker
Agregar tu usuario al grupo docker para no tener que usar sudo en cada comando
sudo usermod -aG docker $USER # Aplicar el cambio en la sesión actual (sin cerrar sesión) newgrp docker
💡 Después de agregar tu usuario al grupo docker, cierra y vuelve a abrir la terminal para que el cambio tome efecto de forma permanente.

Gestión de Imágenes
docker pull · images · rmi
Ver todas las imágenes descargadas localmente con su tamaño, ID y fecha
docker images
Descargar una imagen desde Docker Hub sin ejecutar ningún contenedor
# Formato: docker pull nombre:version docker pull ubuntu docker pull ubuntu:22.04 # versión específica docker pull kalilinux/kali-rolling
Eliminar una imagen local. La imagen no debe estar en uso por ningún contenedor
# Por nombre docker rmi ubuntu # Por ID de imagen docker rmi a1b2c3d4e5f6
Buscar imágenes disponibles en Docker Hub directamente desde la terminal
docker search ubuntu

Gestión de Contenedores
run · stop · rm · exec
Listar contenedores: solo los activos, o todos incluyendo los detenidos
# Solo contenedores corriendo docker ps # Todos los contenedores (activos y detenidos) docker ps -a
Ejecutar un contenedor interactivo — entras directo a la terminal del contenedor
# -i = interactivo | -t = terminal (tty) docker run -it ubuntu bash
Ejecutar un contenedor que se elimina automáticamente al salir — ideal para pruebas rápidas
docker run --rm -it ubuntu bash
Ejecutar un contenedor en segundo plano (modo daemon) con nombre personalizado
# -d = detached (segundo plano) | --name = nombre del contenedor docker run -d -it --name mi-contenedor ubuntu bash
Entrar a un contenedor que ya está corriendo en segundo plano
# Por ID del contenedor docker exec -it <container_id> bash # Por nombre docker exec -it mi-contenedor bash
Detener, iniciar y reiniciar contenedores
docker stop <container_id> # detener docker start <container_id> # iniciar de nuevo docker restart <container_id> # reiniciar
Eliminar contenedores. Debe estar detenido antes de eliminar
# Eliminar uno docker rm <container_id> # Forzar eliminación aunque esté corriendo docker rm -f <container_id> # Eliminar todos los contenedores detenidos docker container prune
Ver información detallada de un contenedor: IP, volúmenes, variables de entorno y más
docker inspect <container_id> # Filtrar solo la IP docker inspect <container_id> | grep IPAddress
Ver los logs (salida) de un contenedor en segundo plano
docker logs <container_id> # Seguir los logs en tiempo real docker logs -f <container_id>

Redes en Docker
network · inspect · labnet
Listar, crear e inspeccionar redes Docker
# Ver redes existentes docker network ls # Crear una red personalizada docker network create labnet # Ver detalles de una red docker network inspect labnet
Conectar contenedores a una misma red personalizada para que se comuniquen entre sí — ideal para laboratorios de hacking
# Correr la víctima en la red labnet docker run -d --name victima --network labnet tleemcjr/metasploitable2 # Correr el atacante en la misma red docker run -it --name atacante --network labnet kalilinux/kali-rolling bash
Explorar la red desde dentro del contenedor
# Ver interfaces y direcciones IP ip a # Ver tabla de rutas ip route # Ver hostname del contenedor hostname
Eliminar una red personalizada (debe estar vacía — sin contenedores conectados)
docker network rm labnet
ℹ️ Los contenedores en la misma red personalizada pueden comunicarse usando su nombre como hostname. Por ejemplo: ping victima desde el contenedor atacante.

Kali Linux en Docker
kali-rolling · Herramientas
Descargar y ejecutar Kali Linux completo en un contenedor interactivo
docker pull kalilinux/kali-rolling docker run -it --name kali kalilinux/kali-rolling bash
Actualizar el sistema e instalar herramientas de red esenciales dentro del contenedor
# Actualizar repositorios y paquetes apt update && apt upgrade -y # Instalar herramientas de red esenciales apt install -y nmap net-tools iputils-ping curl wget \ netcat-openbsd traceroute dnsutils
Instalar herramientas específicas de seguridad y análisis
# Herramientas de análisis de red y seguridad apt install -y nmap wireshark netcat # Metasploit Framework apt install -y metasploit-framework # Suite completa de herramientas Kali apt install -y kali-tools-top10
Verificar el entorno del sistema operativo dentro del contenedor
# Ver información del sistema operativo cat /etc/os-release # Ver IP asignada al contenedor ip a # Ver hostname hostname

Laboratorio — Metasploitable2
Máquina vulnerable
Descargar y correr Metasploitable2 en segundo plano como máquina víctima
# Descargar la imagen vulnerable docker pull tleemcjr/metasploitable2 # Correr en segundo plano con nombre "victima" docker run -d --name victima tleemcjr/metasploitable2
Verificar que está activa y obtener su dirección IP para atacar
# Confirmar que el contenedor está corriendo docker ps # Obtener la IP asignada a la víctima docker inspect victima | grep IPAddress
Montar el laboratorio completo: víctima y atacante en la misma red aislada
# 1. Crear red de laboratorio aislada docker network create labnet # 2. Iniciar la víctima en esa red docker run -d --name victima --network labnet tleemcjr/metasploitable2 # 3. Iniciar el atacante (Kali) en la misma red docker run -it --name atacante --network labnet kalilinux/kali-rolling bash # 4. Desde el atacante, verificar conectividad ping victima nmap -sV victima
⛔ Metasploitable2 es una máquina intencionalmente vulnerable. Úsala solo en redes locales aisladas de laboratorio. Nunca la expongas a Internet.

Tips, Atajos y Limpieza
Atajos · prune · attach
Salir sin detener el contenedor — atajo de teclado para "desconectarse" sin matar el proceso
# Dentro del contenedor, presiona: Ctrl + P → luego Ctrl + Q # Volver a conectarte al mismo contenedor después docker attach <container_id_o_nombre>
Diferencia entre attach y exec para entrar a un contenedor
# attach: reconecta al proceso principal del contenedor docker attach <id> # exec: abre un NUEVO proceso bash dentro del contenedor (recomendado) docker exec -it <id> bash
Limpieza general — eliminar todo lo que no esté en uso
# Eliminar contenedores detenidos docker container prune # Eliminar imágenes sin usar docker image prune # Eliminar redes sin usar docker network prune # Limpieza total (contenedores, imágenes, redes y caché) docker system prune -a
Ver uso de recursos de los contenedores activos en tiempo real
docker stats
Copiar archivos entre el sistema anfitrión y un contenedor
# Del contenedor al host docker cp <id>:/ruta/archivo ./destino/ # Del host al contenedor docker cp ./archivo.txt <id>:/ruta/destino/
💡 Prefiere siempre docker exec -it <id> bash sobre docker attach para entrar a un contenedor. Con exec abres una nueva sesión independiente y no corres el riesgo de detener el proceso principal.
